## INTEGRATION RULES

### Rule 1: No Direct Transitions
**Statement:** All apex transitions must flow through META_APEX  
**Enforcement:** Apex entry points reject non-META_APEX requests  
**Violation:** Access denied, audit generated

### Rule 2: Bidirectional Consistency
**Statement:** Every ascent path has a corresponding descent path  
**Enforcement:** Integration path validation during system initialization  
**Violation:** System initialization blocked until paths defined

### Rule 3: Audit Trail Mandatory
**Statement:** Every apex integration flow generates audit artifact  
**Enforcement:** Audit generation is part of flow completion  
**Violation:** Flow marked incomplete until audit generated

### Rule 4: Coherence Preservation
**Statement:** Integration flows must preserve cross-layer coherence  
**Enforcement:** Coherence check before and after each flow  
**Violation:** Flow aborted, rollback to pre-flow state

## FAILURE MODES

### Failure Mode 1: Unauthorized Access Attempt
**Trigger:** Operation attempts apex access without META_APEX authorization  
**Response:** Access denied, operation returned to origin, audit generated  
**Recovery:** Request META_APEX mediation, retry with authorization

### Failure Mode 2: Coherence Violation During Flow
**Trigger:** Flow execution causes cross-layer incoherence  
**Response:** Flow aborted, rollback to pre-flow state, coherence report generated  
**Recovery:** Identify coherence issue, resolve, retry flow

### Failure Mode 3: Incomplete Flow (Missing Return Path)
**Trigger:** Flow reaches apex but cannot return to origin  
**Response:** Flow marked incomplete, operation stuck in apex layer  
**Recovery:** Define return path, complete flow manually, update documentation

### Failure Mode 4: META_APEX Mediation Failure
**Trigger:** META_APEX unavailable or unresponsive during flow  
**Response:** Flow queued, operation suspended pending META_APEX recovery  
**Recovery:** Restore META_APEX, resume queued flows
