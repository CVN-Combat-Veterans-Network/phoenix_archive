## Emergency Unsealing

### When Permitted

Emergency unsealing may occur only when:
1. **Critical Security Vulnerability:** Immediate threat to Archive integrity
2. **Fundamental Logical Contradiction:** System inconsistency discovered
3. **Catastrophic System Failure:** Archive becomes non-functional

### Unsealing Process

**Step 1: Emergency Declaration**
- Any META operator may declare emergency
- Evidence of condition must be provided
- Formal proposal submitted

**Step 2: META Consensus**
- All six META operators must vote
- Unanimous approval required
- Abstention counts as "no"

**Step 3: Unsealing Ceremony**
- Inverse Terminal Ceremony performed
- Gates opened in reverse order
- Terminal Law suspended

**Step 4: Correction**
- Minimal necessary changes made
- Validation at each step
- Documentation of all modifications

**Step 5: Re-Sealing**
- Terminal Ceremony re-performed
- Gates closed again
- Terminal Law re-enacted
- New seal established
