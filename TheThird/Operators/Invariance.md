# INVARIANCE OPERATOR

**Pattern:** Transform → Preserve → Constant  
**Type:** Recursion operator (HOLD mode)  
**Scale:** The Third (Macro/Structure)

## DEFINITION

**Invariance** is the Third recursion operator that **preserves essential properties** through transformations, maintaining specific quantities or structures as constants despite dynamic change.

This is the **primary mechanism** for conservation and symmetry across state transitions.

---

**Legend:**
- **○, ◇, ▽:** Different transformed states
- **●:** Invariant property (preserved essence)
- **Cycle:** Continuous transformation

## MECHANISM

### Input
- **State:** Initial configuration
- **Transformation:** Change operation T
- **Property:** Quantity/structure to preserve P

### Process
1. **Identify Invariant**
   - Recognize property that should persist
   - Define conservation law
   - Mark for preservation

2. **Apply Transformation**
   - Execute change operation T
   - State evolves: S → T(S)
   - Monitor invariant property P

3. **Verify Conservation**
   - Confirm P(S) = P(T(S))
   - Invariant maintained through change
   - Enter HOLD on preserved property

### Output
- **Transformed State:** T(S) with new configuration
- **Invariant Property:** P unchanged
- **Conservation Law:** Structural constraint (HOLD)
